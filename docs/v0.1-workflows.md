# clew v0.1 Workflows

clew v0.1 is CLI-first and correctness-first. Filesystem skill bundles are canonical; SQLite is derived state that can be deleted and rebuilt.

## Bundle Format

A canonical bundle is a directory with:

- `clew.yaml`
- `skill.md`

Optional preserved directories are `examples/`, `templates/`, `assets/`, and `tests/`. clew preserves these files but does not execute them.

## Registry

Registry precedence is:

1. `session`
2. `project`
3. `org`
4. `global`

Project discovery reads `project/.clew/` and the repository `skills/` reference bundles. Global discovery reads `~/.clew/global/`. Organization discovery reads `~/.clew/orgs/<org>/` when an org is provided to the core API.

SQLite registry state is stored in `.clew-registry.db` by the CLI. The preferred driver is `better-sqlite3`; if its native binding cannot load on the current Node/runtime combination, clew falls back to Node's built-in SQLite API. In both cases, filesystem bundles remain canonical and the registry index is rebuildable.

Registry rebuilds degrade invalid filesystem bundles instead of dropping the whole registry read. `loadSkillBundle(directory)` remains strict for direct callers, but registry discovery catches schema-invalid candidate directories, loads the remaining valid bundles, and returns one `skill_bundle_invalid` compatibility warning per invalid bundle.

## CLI Examples

```sh
clew list
clew search refactor
clew lookup refactor-safely
clew recommend "refactor TypeScript cleanup"
clew explain refactor-safely "refactor TypeScript cleanup"
clew overlaps
clew conflicts
clew telemetry
clew doctor
```

Read-oriented commands return stable JSON envelopes for script consumers. This is a v0.1 pre-release breaking contract cleanup: commands that previously returned raw arrays now return named top-level fields plus `warnings`.

Registry-backed read commands such as `list`, `search`, `lookup`, `recommend`, `explain`, `overlaps`, `conflicts`, and `doctor` include registry degradation warnings in their `warnings` arrays. A malformed local bundle appears as an error-severity warning instead of a top-level failure envelope:

```json
{
  "skills": [{ "id": "typescript-core" }],
  "warnings": [
    {
      "code": "skill_bundle_invalid",
      "severity": "error",
      "field": "/project/skills/future-kind",
      "message": "manifest.kind [invalid_enum_value]: Invalid enum value. Expected 'instruction_skill', received 'workflow_skill'"
    }
  ]
}
```

`clew list`:

```json
{ "skills": [], "warnings": [] }
```

`clew search refactor`:

```json
{ "query": "refactor", "skills": [], "warnings": [] }
```

`clew lookup refactor-safely`:

```json
{ "skillId": "refactor-safely", "bundle": { "manifest": { "id": "refactor-safely" } }, "warnings": [] }
```

Unknown or disabled skills return `bundle: null` with an explicit warning:

```json
{ "skillId": "refactor-safely", "bundle": null, "warnings": [{ "code": "skill_unknown" }] }
```

`clew recommend "refactor TypeScript cleanup"`:

```json
{ "query": "refactor TypeScript cleanup", "recommendations": [], "warnings": [] }
```

`clew explain refactor-safely "refactor TypeScript cleanup"`:

```json
{ "skillId": "refactor-safely", "query": "refactor TypeScript cleanup", "recommendation": { "skillId": "refactor-safely" }, "warnings": [] }
```

Missing, disabled, or unrecommended skills return `recommendation: null` with `skill_unknown`, `skill_disabled`, or `skill_not_recommended`:

```json
{ "skillId": "refactor-safely", "query": "refactor TypeScript cleanup", "recommendation": null, "warnings": [{ "code": "skill_not_recommended" }] }
```

`clew overlaps`:

```json
{ "overlaps": [], "warnings": [] }
```

`clew conflicts`:

```json
{ "conflicts": [], "warnings": [] }
```

`doctor` keeps its diagnostic fields and always includes `warnings`, including registry degradation warnings and AGENTS.md diagnostics.

Provider interop commands accept JSON input for import and produce scriptable JSON output for export:

```sh
clew import claude ./skill.json
clew import opencode ./agent.json
clew export claude refactor-safely
clew export opencode refactor-safely
```

## Warning Model

clew preserves unknown provider metadata under `extensions.<provider>`. Unsupported or lossy semantics produce compatibility warnings instead of silent omission. Registry degradation warnings use the same warning shape; invalid canonical bundles use `severity: "error"` because the bundle could not be indexed, even though the registry read continues for valid bundles.

## Non-Goals

v0.1 does not execute workflows, run autonomous activation, install remote registries, use embeddings, run a daemon, or inject prompts implicitly. MCP support is intentionally read-oriented: search, recommend, explain, and lookup.
